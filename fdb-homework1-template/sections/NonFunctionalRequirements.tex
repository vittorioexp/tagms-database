\section{Non Functional Requirements}
%Non functional requirements: what other characteristics should the system have, beside functional requirements?
The DBMS should have the following non-functional requirements:
\begin{itemize}
    \item It should store data following the GDPR (e.g customer’s data).
    \item It should be scalable to be able to work with big amounts of data.
    \item It should have a back-up of the data to avoid loos of it if SW/HW malfunctioing happens.
    \item It should guarantee privacy ot the data using authorisation and authentication to access the system.
    \item It should be easy to use.
\end{itemize}
